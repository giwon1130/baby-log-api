## ADR-001 — Spring Boot + Kotlin (2025-Q4)

**문맥**: 단순 CRUD + 약간의 비즈니스 로직. 1인 개발.

**대안**:
- Node + Express — 빠르지만 타입 시스템 약함
- Spring Boot + Java — 익숙하지만 보일러플레이트
- **Spring Boot + Kotlin** — 익숙 + null 안전성 + data class

**결정**: Kotlin 채택.

## ADR-002 — JdbcTemplate (JPA 미사용)

**문맥**: 도메인 단순 (모두 baby_id로 묶이는 단순 CRUD). JPA 추상화 큰 이득 X.

**대안**:
- Spring Data JPA — 자동 쿼리 / 마이그레이션
- Spring Data JDBC — 가벼운 ORM
- **JdbcTemplate** — 명시적 SQL

**결정**: JdbcTemplate.

**근거**:
- 작은 도메인 — JPA의 cascade / dirty checking 같은 기능 안 필요
- SQL이 명시적 → 성능 예측 / 디버깅 쉬움
- `bl_cry_samples` 의 음향 feature 컬럼 추가 같은 ad-hoc 변경에 유연

**트레이드오프**: 도메인 커지면 (10+ 테이블) JPA 검토. 현재 7개라 OK.

## ADR-004 — 스키마 마이그레이션: `if not exists` 만 (Flyway 미도입, 2025-Q4)

**문맥**: 초기 단순함 vs Flyway 도입 비용.

**대안**:
- Flyway — 정식 마이그레이션 도구. 버전 관리.
- Liquibase — 동상.
- **`@PostConstruct + create table if not exists / alter add column if not exists`** — 자체

**결정**: 자체 방식 — `bootstrap/DatabaseConfig.SchemaInitializer.init()` 한 곳에 모든 DDL.

**근거**:
- 1인 개발 + 단일 환경 (현재 prod 하나)
- 테이블 추가 / 컬럼 추가만 자주 — 둘 다 `if not exists` 로 안전
- Flyway 마이그레이션 파일 관리 부담 > 이득

**트레이드오프**:
- 컬럼 삭제 / 타입 변경 → 수동 SQL (Railway 콘솔)
- 마이그레이션 이력 추적 안 됨 — 로그로만

**도입 트리거**: 운영 DB 컬럼 삭제 1번이라도 필요하면 즉시 Flyway 검토.

## ADR-006 — 인증 미도입 (잠정, 2026-04)

**문맥**: 1차 출시 사용자 = 본인 가족 + 지인. 작은 범위.

**결정**: 인증 안 만듦. `babyId` 만으로 접근.

**리스크**:
- URL 노출 시 데이터 접근 가능 (실제로 URL이 외부로 새는 경로는 거의 없음)
- 로그 / 분석 도구가 URL 캡처할 가능성

**완화**:
- HTTPS 필수 (Railway 자동)
- API 호출 로그에 babyId 마스킹 (현재 X — 운영 시 추가)

**도입 트리거**: 외부 사용자 100명 초과 또는 데이터 유출 사고.

도입 옵션은 [`운영/보안.md`](../운영/보안.md) 참고.

## ADR-007 — 음향 feature 확장 (2026-04)

**문맥**: Phase 1에서 음향 정보가 cry_confidence + 볼륨만. "왜 그렇게 판단했는지"
사용자에게 설명할 데이터 부족.

**대안**:
- (A) MFCC 13~20개 추출 + 모델 학습 — 데이터 부족으로 실패
- (B) 핵심 acoustic descriptor (피치 / ZCR / 리듬성) 추출 + 휴리스틱 규칙
- (C) 그대로 두고 사용자 정정에 의존

**결정**: **(B)**.

**구현**:
- iOS Accelerate (`vDSP_dotpr`) 기반 autocorrelation 피치 검출
- RMS envelope 자기상관으로 rhythmicity
- ZCR 단순 루프
- 백엔드 분류 규칙: 피치 > 600Hz → PAIN, 리듬 > 0.45 → HUNGER 등

**결과**: 결과 화면에 "분석 디테일" 섹션 추가 가능 — 사용자 신뢰도 ↑
