## 한 장 그림

```
[Expo App] ──HTTPS──> [Railway: Spring Boot 3.3 / Kotlin] ──> [Railway PostgreSQL]
                                  │
                                  └─ JdbcTemplate (JPA 미사용)
```

## 책임 분리

- **Controller**: HTTP 매핑 + 입력 검증 (얇음). DTO 변환 후 Service 위임.
- **Service**: 도메인 로직 + JdbcTemplate 직접 호출. 트랜잭션 단위.
- **Schema**: `DatabaseConfig.SchemaInitializer.init()` 한 곳에서 DDL 실행.
- **DTO**: 각 features 폴더 안에 `data class`. 별도 `dto` 패키지 없음.

JPA Entity / Repository 미사용 — JdbcTemplate으로 SQL 명시. 이유는
[`의사결정로그.md`](의사결정로그.md).

## 도메인 의존성

```
Family ──┐
         ├─→ Baby ──→ Feed / Diaper / Sleep / GrowthRecord / CrySample
         │           
         └─→ (향후) FamilyMember (인증 도입 시)
```

- `Family` 1 : N `Baby`
- `Baby` 1 : N 모든 기록 도메인
- `bl_cry_samples` 만 약간 특수: `predicted_label` + `confirmed_label` 두 컬럼,
  컨텍스트 스냅샷 (수유 경과 등) 함께 저장

## 데이터 흐름 — 울음 분석 (가장 복잡)

```
POST /api/v1/babies/:id/cry-samples (음향 feature 페이로드)
  │
  └─ CryAnalysisController.submit
       └─ CryAnalysisService.submit
            ├─ buildContext(babyId)              # 마지막 수유/기저귀/수면 + 일령 + 시각
            ├─ classify(features, context, history)
            │    ├─ apply context priors          # 시간 기반 점수
            │    ├─ apply 음향 규칙               # 피치 / 리듬 / ZCR
            │    ├─ if confirmedCount >= 20:
            │    │    └─ similarity 부스트 (euclidean)
            │    └─ return CryPrediction[] (정렬)
            └─ INSERT bl_cry_samples (예측 + feature + context)
            └─ 응답 CrySampleResponse (predictions + learningStage)

PATCH /api/v1/cry-samples/:id/confirm { confirmedLabel }
  └─ UPDATE bl_cry_samples SET confirmed_label = ?, confirmed_at = now()
  └─ 응답 (갱신된 learningStage 포함)
```

## 외부 의존성

- `org.springframework.boot:spring-boot-starter-web`
- `org.springframework:spring-jdbc`
- `org.postgresql:postgresql`
- (향후) `spring-security` — 인증 도입 시
- (향후) `flyway-core` — 스키마 마이그레이션 정식화 시

## 트랜잭션 / 동시성

- `@Transactional` 미사용 — 단일 INSERT/UPDATE 위주라 큰 문제 없음
- 가족당 동시 사용자 ~3명 가정 → 충돌 거의 없음
- `bl_sleep_records` 의 진행 중 세션은 `wokeAt IS NULL`로 한 번에 1개만 허용 (앱 측 검증)
- 향후 다중 사용자 시 optimistic locking 검토

## 핵심 설계 결정 (요약)

자세한 건 [`의사결정로그.md`](의사결정로그.md):

1. **JPA → JdbcTemplate** — 단순 도메인, 마이그레이션 명시성
2. **DDL `if not exists` + ALTER** — 정식 마이그레이션 도구 미사용 (Flyway 보류)
3. **CryAnalysisService 모놀리식** — 규칙 늘어나면 데이터화
4. **Railway 채택** — fly.io에서 이전, PG plugin이 더 매끄러움
