# 데이터 모델 (DB 스키마) ★ SoT

> **누가 봄**: BE / FE / DBA
> **언제 봄**: 새 테이블/컬럼 추가, 의미 확인
> **이 문서가 SoT**. 클라이언트 측 [데이터모델](https://github.com/giwon1130/baby-log-app/tree/main/docs/엔지니어링/데이터모델.md) 은 이걸 기반으로 한 TS 타입.

## 테이블 목록

| 테이블 | 역할 | 외래키 |
|--------|------|--------|
| `bl_families` | 가족 (초대 코드) | — |
| `bl_babies` | 아기 | family_id |
| `bl_feed_records` | 수유 | baby_id |
| `bl_diaper_records` | 기저귀 | baby_id |
| `bl_sleep_records` | 수면 | baby_id |
| `bl_growth_records` | 성장 측정 | baby_id |
| `bl_cry_samples` | 울음 분석 샘플 | baby_id |

모든 ID는 `varchar(36)` UUID. 시간은 `timestamptz`.

## 상세

### bl_families
```sql
create table bl_families (
    id            varchar(36) primary key,
    invite_code   varchar(16) not null unique,
    created_at    timestamptz not null default now()
)
```
- `invite_code`: 영숫자 8자리 (현재 — 길이 가변)
- 한 가족당 여러 아기 가능 (현재 UI는 첫 아기만)

### bl_babies
```sql
create table bl_babies (
    id              varchar(36) primary key,
    family_id       varchar(36) not null references bl_families(id),
    name            varchar(50) not null,
    birth_date      date not null,
    gender          varchar(10) not null,    -- 'MALE' | 'FEMALE'
    birth_weight_g  integer,
    birth_height_cm double precision,
    created_at      timestamptz not null default now()
)
```
- `daysOld` 는 응답에서 계산 (저장 안 함)
- 삭제 시 cascade 안 됨 — 직접 SQL로 종속 데이터 정리 필요

### bl_feed_records
```sql
create table bl_feed_records (
    id             varchar(36) primary key,
    baby_id        varchar(36) not null references bl_babies(id),
    fed_at         timestamptz not null,
    amount_ml      integer not null,
    feed_type      varchar(20) not null default 'FORMULA',  -- FORMULA|BREAST|MIXED
    note           text not null default '',
    left_minutes   integer,
    right_minutes  integer
)
create index idx_feed_baby_fed on bl_feed_records(baby_id, fed_at desc)
```
- `next_feed_at`은 응답에서 계산 (저장 안 함) — 일령별 권장 간격으로

### bl_diaper_records
```sql
create table bl_diaper_records (
    id          varchar(36) primary key,
    baby_id     varchar(36) not null references bl_babies(id),
    changed_at  timestamptz not null,
    diaper_type varchar(10) not null,    -- WET|DIRTY|MIXED|DRY
    note        text not null default ''
)
create index idx_diaper_baby_changed on bl_diaper_records(baby_id, changed_at desc)
```

### bl_sleep_records
```sql
create table bl_sleep_records (
    id        varchar(36) primary key,
    baby_id   varchar(36) not null references bl_babies(id),
    slept_at  timestamptz not null,
    woke_at   timestamptz,                -- NULL이면 진행 중
    note      text not null default ''
)
create index idx_sleep_baby_slept on bl_sleep_records(baby_id, slept_at desc)
```
- `duration_minutes` 는 응답에서 계산
- 한 아기에 진행 중 (`woke_at IS NULL`) 세션은 1개만 허용 — 앱 측 검증

### bl_growth_records
```sql
create table bl_growth_records (
    id          varchar(36) primary key,
    baby_id     varchar(36) not null references bl_babies(id),
    measured_at timestamptz not null,
    weight_g    integer,
    height_cm   double precision,
    head_cm     double precision,
    note        text not null default ''
)
create index idx_growth_baby_measured on bl_growth_records(baby_id, measured_at desc)
```

### bl_cry_samples (가장 복잡)

```sql
create table bl_cry_samples (
    id                              varchar(36) primary key,
    baby_id                         varchar(36) not null references bl_babies(id),
    recorded_at                     timestamptz not null default now(),
    duration_sec                    double precision not null,

    -- Apple SoundAnalysis + 볼륨 (Phase 1)
    cry_confidence_avg              double precision,
    cry_confidence_max              double precision,
    avg_volume_db                   double precision,
    peak_volume_db                  double precision,

    -- Phase 2A 음향 feature (autocorrelation 기반)
    pitch_mean_hz                   double precision,
    pitch_std_hz                    double precision,
    pitch_max_hz                    double precision,
    voiced_ratio                    double precision,
    zcr_mean                        double precision,
    rhythmicity                     double precision,

    -- 컨텍스트 스냅샷 (분류 시점의)
    minutes_since_last_feed         integer,
    minutes_since_last_diaper       integer,
    minutes_since_last_sleep_start  integer,
    minutes_since_last_sleep_end    integer,
    is_during_sleep                 boolean not null default false,
    baby_age_days                   integer,
    time_of_day_hour                integer,

    -- 예측 + 사용자 확정
    predicted_label                 varchar(20) not null,
    predicted_confidence            double precision not null,
    confirmed_label                 varchar(20),
    confirmed_at                    timestamptz,
    note                            text not null default ''
)

create index idx_cry_baby_recorded on bl_cry_samples(baby_id, recorded_at desc)
create index idx_cry_baby_confirmed on bl_cry_samples(baby_id) where confirmed_label is not null
```

- **음성 파일은 저장 안 함.** 이 테이블이 보관하는 건 숫자 feature + 메타데이터뿐.
- **컨텍스트 스냅샷** 저장하는 이유: 나중에 재분류 / 분석 시 그 시점 정보 필요
- `confirmed_label is not null` 인 것들이 학습 데이터 — partial index로 쿼리 빠름
- Phase 2A에서 6개 컬럼 추가 — `alter table ... add column if not exists` 로 ALTER

## 인덱스 정책

- 모든 `bl_*` 도메인 테이블에 `(baby_id, <시간>desc)` 복합 인덱스
- 부분 인덱스 사용 (`bl_cry_samples` confirmed) — 필터링 강한 쿼리에서 효과적

## 마이그레이션 정책

현재 Flyway / Liquibase 미사용. `DatabaseConfig.SchemaInitializer.init()` 의
`@PostConstruct` 에서 `create table if not exists` / `alter table add column if not exists`
실행.

장단점:
- 장점: 단순. 별도 도구 학습 X.
- 단점: 컬럼 삭제 / 타입 변경 자동 안 됨. 인덱스 정리도 수동.

도입 트리거: 운영 DB 컬럼 삭제 1회 발생 시 즉시 Flyway 검토.
