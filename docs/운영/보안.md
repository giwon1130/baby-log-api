## 인증 (현재)

**없음.** URL의 `babyId` / `familyId` 를 알면 누구나 접근.

위험과 완화 → [`엔지니어링/의사결정로그.md` ADR-006](../엔지니어링/의사결정로그.md#adr-006--%EC%9D%B8%EC%A6%9D-%EB%AF%B8%EB%8F%84%EC%9E%85-%EC%9E%A0%EC%A0%95-2026-04).

도입 트리거:
- 외부 사용자 100명 초과
- 멤버 회수 / 추방 기능 요구
- 데이터 유출 사고

도입 옵션 (우선순위):
1. **Firebase Auth + 백엔드 token verify** (빠름)
2. 자체 JWT (Spring Security)
3. Apple Sign-In + 자체 토큰

## CORS

현재 별도 설정 없음 — Spring 기본 (모든 origin 차단). 클라이언트는 Expo / native라
브라우저 CORS 영향 없음.

웹 클라이언트 도입 시:
```kotlin
@Configuration
class CorsConfig : WebMvcConfigurer {
    override fun addCorsMappings(registry: CorsRegistry) {
        registry.addMapping("/api/**")
            .allowedOrigins("https://app.babylog.kr")  // 화이트리스트만
            .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE")
    }
}
```

## Rate limiting

**없음.** 1인/소수 사용자라 필요 X. 향후 도입:
- Bucket4j 또는 Resilience4j
- 가족당 분당 60회 / 동일 IP 분당 120회

## SQL Injection

JdbcTemplate `?` 파라미터 바인딩만 사용. 문자열 concat SQL 없음 (검토 시 grep
`"select.*\\$\\{`).

## 데이터 보관

| 데이터 | 보관 위치 | 백업 |
|--------|----------|------|
| 가족/아기/기록 | Railway PG | Railway 자동 일일 백업 (무료 티어 7일 보관) |
| 음성 파일 | **저장 안 함** | — |
| 음향 feature (숫자) | bl_cry_samples | DB 백업 포함 |

복구 절차:
```bash

# 또는
railway run pg_restore --dbname=$DATABASE_URL backup.dump
```

## 로그 정책

- **PII 로그 금지** — 아기 이름, 메모, 가족 코드 raw
- ID는 마스킹 권장 (예: `bb_a1b2`)
- Railway 로그는 30일 보관 — 그 이상 필요하면 외부로 export
