| 변수 | 필수 | 기본값 | 설명 |
|------|------|--------|------|
| `BABY_LOG_DB_URL` | ✅ | `jdbc:postgresql://localhost:5492/babylog` | JDBC URL |

## Railway

### `DATABASE_URL` → 분해 매핑

Railway PostgreSQL plugin은 `DATABASE_URL` 한 줄로 주입:
```
postgresql://postgres:****@containers-us-west-XXX.railway.app:7234/railway
```

이걸 Spring에서 그대로 못 씀 (JDBC URL 형식 아님). **세 변수로 분해:**

```
BABY_LOG_DB_URL      = jdbc:postgresql://${PGHOST}:${PGPORT}/${PGDATABASE}?sslmode=require
BABY_LOG_DB_USERNAME = ${PGUSER}
BABY_LOG_DB_PASSWORD = ${PGPASSWORD}
```

Railway는 `${{Postgres.PGHOST}}` 같은 cross-service reference 지원 —
"Reference" 버튼으로 추가 가능.

또는 수동 입력 (DB 비밀번호 변경 시 수동 갱신 필요).

## 시크릿 회전

| 시크릿 | 회전 주기 | 절차 |
|--------|----------|------|
| `BABY_LOG_DB_PASSWORD` | 6개월 또는 사고 시 | Railway DB Settings → Reset password → API 서비스 ENV 갱신 |
| (향후) APNs Auth Key | 분실 / 유출 시 즉시 | Apple Developer → Keys → Revoke → 새로 발급 |

## 향후 추가 예정

| 변수 | 용도 |
|------|------|
| `BABY_LOG_JWT_SECRET` | 인증 도입 시 |
| `BABY_LOG_APNS_KEY_ID` | 푸시 알림 |
| `BABY_LOG_APNS_TEAM_ID` | 동상 |
| `SENTRY_DSN` | 에러 추적 |

## 로컬 vs 운영

| 항목 | 로컬 | 운영 (Railway) |
|------|------|---------------|
| DB | `localhost:5492` (docker-compose) | Railway PG (containers-us-west) |
| Port | 8092 | Railway가 `$PORT` 주입 — Spring이 자동 사용 |
| HTTPS | X (http) | ✅ Railway 자동 |
| ENV | `application.yml` 또는 `BABY_LOG_DB_*` 직접 export | Railway Variables |

> ⚠️ Railway는 `$PORT` 동적 — `SERVER_PORT` 명시하지 말고 Spring이 알아서 잡게 두기.

## 보안

- 시크릿 절대 git에 커밋 X
- `application.yml` 의 기본값은 로컬 개발용 비밀 — 운영에는 Railway ENV 우선
- PR 리뷰 시 `BABY_LOG_DB_*` 검색해서 하드코딩 없는지 확인
