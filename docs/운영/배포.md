# 배포 (서버) ★ SoT

> **누가 봄**: BE / DevOps
> **언제 봄**: 새 배포 / 사고 / 환경 셋업
> **이 문서가 SoT**. 클라이언트 배포는 [앱 저장소 배포](https://github.com/giwon1130/baby-log-app/tree/main/docs/운영/배포.md).

## 현황

- **호스팅**: [Railway](https://railway.app) (무료 티어)
- **트리거**: GitHub `main` 브랜치 푸쉬 → Railway가 자동 빌드/배포 (Nixpacks 자동 감지)
- **DB**: Railway PostgreSQL plugin (같은 프로젝트 내)

## 일상 배포 (자동)

```bash
git push origin main
```

Railway 대시보드에서 빌드 로그 확인. 보통 2~3분 안에 배포 완료.

## 첫 셋업 (참고용 — 이미 끝남)

### 1. Railway 프로젝트 만들기
1. https://railway.app → New Project → Deploy from GitHub
2. `giwon1130/baby-log-api` 선택 → Kotlin/Gradle 자동 감지
3. 첫 배포는 환경변수 없어서 실패 — 정상

### 2. PostgreSQL plugin
1. 프로젝트 → New → Database → PostgreSQL
2. 자동으로 `DATABASE_URL` env가 같은 프로젝트의 다른 서비스에 주입됨

### 3. 환경변수 매핑

`DATABASE_URL` 형식: `postgresql://user:pass@host:port/db`

이걸 Spring 이 쓰는 형식으로 분해해서 등록:

```
BABY_LOG_DB_URL       = jdbc:postgresql://<host>:<port>/<db>?sslmode=require
BABY_LOG_DB_USERNAME  = <user>
BABY_LOG_DB_PASSWORD  = <pass>
```

(Railway는 `${{Postgres.DATABASE_URL}}` 같은 reference로 가능 —
자세한 건 [`환경변수.md`](환경변수.md))

### 4. 도메인
Settings → Networking → Generate Domain → `<random>.up.railway.app`

원하면 커스텀 도메인 (CNAME) 추가 가능.

## 빌드 설정

Railway가 Nixpacks 으로 자동 감지:
- `gradlew bootJar` 빌드
- `java -jar build/libs/*.jar` 실행

추가 설정 필요 없음.

명시적으로 하고 싶으면 `railway.json` 또는 `Procfile`:
```
web: java -jar build/libs/baby-log-api-0.0.1-SNAPSHOT.jar
```

## 로컬 개발

### Docker Compose (PostgreSQL 같이)
```bash
cd ../baby-log    # 모노레포 디렉토리 (외부)
docker-compose up
```

### 단독 실행
```bash
# PostgreSQL 별도 띄워두고
./gradlew bootRun
```

기본:
- Server: `http://localhost:8092`
- DB: `localhost:5492` (docker-compose 기준), 로컬 PG는 `5432` 표준

## 배포 검증

```bash
# Health check (Spring boot actuator 미도입 — 단순 endpoint로)
curl https://<도메인>/api/v1/families -X POST
# → { "id": "...", "inviteCode": "..." } 받으면 OK
```

## 비상 롤백

### 옵션 A: Railway 대시보드
Deployments 탭 → 이전 successful 배포 → "Redeploy"

### 옵션 B: Git revert
```bash
git revert <bad-commit>
git push origin main
```
자동 재배포 트리거됨.

## 장애 대응

[`장애대응.md`](장애대응.md) 참고. 흔한 사고:
- Cold start (2~3초) — 무료 티어 기본
- DB 연결 풀 고갈 — 동시 사용자 급증 시
- ENV 변경 후 재배포 안 됨 — Railway는 ENV 바뀌면 자동 재시작 됨, 안 되면 수동 redeploy
