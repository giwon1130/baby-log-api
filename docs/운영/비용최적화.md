## 현재

**Railway 무료 티어 ($5/월 크레딧 내).**

| 자원 | 사용 | 한도 |
|------|------|------|
| 메모리 | ~256MB | 무료 티어 한도 OK |
| CPU | 거의 idle | OK |
| DB 스토리지 | < 100MB | 1GB 한도 OK |
| 트래픽 | 월 < 1GB | OK |

## 사용자 100명까지 가는 길

추정:
- 가족당 일일 행 수: ~30 (수유 + 기저귀 + 수면 + 울음)
- 100가족 × 30 × 365일 ≈ 110만 행/년
- 평균 행 크기 ~200 bytes → ~220MB/년
- DB는 충분 (1GB 무료 한도 내)

API 호출 빈도: 가족당 일 100회 → 100가족 × 100 × 30일 = 30만/월. Railway 무료 한도
충분.

**병목 예상**:
1. **Cold start** — 무료 티어 sleep. 사용자 경험 저하 시 유료 ($5+/월)
2. **메모리** — 동시 접속 ~50명 넘으면 GC 압박. 인스턴스 크기 ↑

## 비용 절감 원칙

### 인프라
- **무료 티어 최대 활용** — Railway, Cloudflare, GitHub Actions
- 도메인 / CDN 도입은 사용자 100명 임계값 후
- 모니터링 SDK (Sentry 등) 무료 한도 사용

### 쿼리 / DB
- N+1 회피 — 통계 endpoint 처럼 일괄 SELECT로 가져오기
- 인덱스 누락 정기 점검 (`pg_stat_user_indexes`)
- 큰 데이터 (음성/이미지) 절대 DB에 안 넣기 — 베이비로그는 해당 없음 ✅

### 외부 API
- 현재 0 — 모든 추론 클라이언트 (CoreML)
- 향후 ML 서버 추론 검토 시 GPU 비용 발생 → 신중

### 트래픽
- API 응답 압축 (Spring 자동 gzip 활성화 — `application.yml`)
- 페이지네이션 — `limit` 기본값 50 (현재 OK)

## 비용 단계 예측

| 단계 | 사용자 | 월 비용 | 액션 |
|------|--------|---------|------|
| 0 ~ 50명 | 본인 + 지인 | $0 | 현재 |
| 50 ~ 200명 | 베타 / TestFlight | $5 ~ $10 | Railway 유료 (always-on) |
| 200 ~ 1000명 | 정식 출시 초기 | $20 ~ $50 | DB 인스턴스 ↑, Sentry 도입 |
| 1000명 + | 본격 운영 | $100+ | 비즈니스 모델 필요 (광고 / 인앱결제) |
